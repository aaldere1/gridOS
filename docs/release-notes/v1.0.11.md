# gridOS 1.0.11

gridOS 1.0.11 is a terminal workflow polish release. It makes copy and paste
between terminal panes behave like a native Mac workspace: select in one pane,
copy, click another pane, paste.

## What's New

- Added shared system-clipboard routing for terminal pane menu commands.
- Updated terminal focus tracking so clicked panes become the active target for
  Copy, Paste, Select All, Clear, Reset, and related Terminal menu actions.
- Preserved SwiftTerm's native paste path so bracketed paste behavior remains
  correct in shells that support it.
- Added tests for pane-to-pane copy/paste, clicked-pane focus activity,
  no-selection clipboard preservation, and empty-clipboard fallback behavior.
- Kept Command-T pane creation, Control-Tab pane traversal, drag-to-rearrange
  pane headers, AI Command Helper screenshot OCR, DeepSeek and xAI provider
  support, and Sparkle automatic updates.

## Why It Stands Out

This release removes a small but important workflow snag. Multi-pane terminals
only feel native when the active pane matches the pane you are actually using,
and clipboard operations now follow that expectation without private buffers or
special app-only paste behavior.

## Install

1. Download `gridOS-1.0.11-19-a3fb5ec.dmg` from the `v1.0.11` GitHub release.
2. Open the DMG.
3. Drag `gridOS.app` into Applications.
4. If you installed an earlier copy, replace it.
5. Launch gridOS from Applications.

If macOS blocks the app, do not bypass Gatekeeper. Report it as a release issue.
