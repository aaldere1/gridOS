# gridOS 1.0.14

gridOS 1.0.14 is a terminal clipboard follow-up release. It fixes the remaining
source-pane shortcut path so Command-A in one terminal pane/session selects that
pane before Command-C copies it for paste into another pane/session.

## What's New

- Command-A now selects text in the terminal pane that emitted the shortcut
  before falling back to the active pane.
- Command-A and Command-C now share the same source-pane routing behavior for
  split terminal panes.
- Added regression coverage for the exact late-review shape: pane A emits
  Select All, pane B is active, and pane A must receive the selection command.
- Retains the v1.0.13 source-pane copy fix and the v1.0.12 cross-pane
  pasteboard bridge.

## Why It Matters

Pane-to-pane copy/paste only feels native when selection and copy both follow
the terminal pane you are using. This release completes that path: select or
Select All in one pane, copy from that pane, move to another pane, and paste.

## Install

1. Download `gridOS-1.0.14-22-c245751.dmg` from the `v1.0.14` GitHub release.
2. Open the DMG.
3. Drag `gridOS.app` into Applications.
4. If you installed an earlier copy, replace it.
5. Launch gridOS from Applications.

If macOS blocks the app, do not bypass Gatekeeper. Report it as a release issue.
