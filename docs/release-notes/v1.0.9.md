# gridOS 1.0.9

gridOS 1.0.9 turns the terminal workspace into a more native Mac command
surface. The release adds fast pane creation, keyboard pane traversal, and
drag-to-rearrange layout control on top of the 1.0.8 terminal polish work.

## What's New

- Added Command-T for creating a new terminal pane to the right of the active
  pane.
- Added Control-Tab and Control-Shift-Tab for cycling through terminal panes
  while preserving normal shell Tab input.
- Added drag-and-drop pane rearrangement from pane headers, including
  before/after/above/below placement.
- Kept active-pane focus explicit after pane creation, movement, and keyboard
  traversal.
- Preserved focused terminal copy, paste, select-all, clear, reset, and run
  behavior across multi-pane layouts.
- Shipped the update as a signed, notarized, stapled DMG and ZIP with Sparkle
  automatic-update metadata.

## Why It Stands Out

The app now behaves more like a native Mac workspace instead of a static
terminal frame. Command-T gets a new shell beside your current work, Control-Tab
moves between panes without stealing shell completion, and dragging pane headers
makes layout changes visible and deliberate.

## Install

1. Download `gridOS-1.0.9-17-2d2fe8d.dmg` from the `v1.0.9` GitHub release.
2. Open the DMG.
3. Drag `gridOS.app` into Applications.
4. If you installed an earlier copy, replace it.
5. Launch gridOS from Applications.

If macOS blocks the app, do not bypass Gatekeeper. Report it as a release issue.
