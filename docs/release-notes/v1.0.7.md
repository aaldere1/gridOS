# gridOS 1.0.7

gridOS 1.0.7 is the HUD and provider expansion release: sharper visual modes,
live terminal font-size control, screenshot-aware Command-K context, and more
direct AI provider choices on top of the signed automatic-update lane.

## What's New

- Added Matrix, Amber CRT, and Redline visual modes alongside Tron, Severance,
  and Apple-native.
- Added a right-rail HUD Signal stack for local host pulse, shell state, visual
  signature, and activity.
- Added a header visual-style cycle control for quick mode changes.
- Added live terminal font-size controls that update existing panes without
  restarting shells.
- Added DeepSeek and xAI as first-class Command Intelligence providers.
- Refined the AI Command Helper sheet with roomier spacing, a scrollable
  compose body, and a pinned action row.
- Added screenshot drops to AI Command Helper. gridOS runs local OCR, shows the
  previewed context, and sends OCR text and metadata only; image pixels and
  local file paths stay on the Mac.
- Kept provider API keys in the macOS Keychain, with provider/model preferences
  stored separately as normal settings.
- Updated public docs to describe the six visual modes and four provider paths.

## Why It Stands Out

This release makes gridOS feel more alive without turning it into a gimmick.
The HUD is denser and cooler, the terminal remains the main event, and Command-K
can use Anthropic, OpenAI, DeepSeek, or xAI without adding a gateway or proxy.
Dropped screenshots become reviewed local OCR context instead of opaque uploads.

## Install

1. Download `gridOS-1.0.7-15-8a1d12e.dmg` from the `v1.0.7` GitHub release.
2. Open the DMG.
3. Drag `gridOS.app` into Applications.
4. If you installed an earlier copy, replace it.
5. Launch gridOS from Applications.

If macOS blocks the app, do not bypass Gatekeeper. Report it as a release issue.
