# gridOS 1.0.13

gridOS 1.0.13 is a terminal clipboard correctness release. It fixes the case
where selected text copied from one terminal pane/session could fail to paste
into another pane/session inside the same gridOS window.

## What's New

- Command-C now copies from the terminal pane that emitted the shortcut before
  falling back to the active pane or other selected panes.
- Command-A, Command-C, Command-V, and Command-T terminal shortcuts now emit
  workspace activity synchronously so selection is read before focus state can
  drift.
- Added regression coverage for the exact failure shape: pane A emits Copy,
  pane B is active, both panes have selections, and pane A's selected text must
  win.
- Verified a live app smoke where text copied from pane 1 pasted into pane 2.

## Why It Matters

Multi-pane terminals only feel native when selection, copy, and paste follow the
pane you are actually using. This release makes cross-pane terminal copy/paste
feel predictable: select in one pane, copy, move to another pane, paste.

## Install

1. Download `gridOS-1.0.13-21-00e2893.dmg` from the `v1.0.13` GitHub release.
2. Open the DMG.
3. Drag `gridOS.app` into Applications.
4. If you installed an earlier copy, replace it.
5. Launch gridOS from Applications.

If macOS blocks the app, do not bypass Gatekeeper. Report it as a release issue.
