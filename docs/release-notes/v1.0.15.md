# gridOS 1.0.15

gridOS 1.0.15 fixes the remaining intermittent multi-pane paste path. Command-V
emitted from a terminal pane now pastes into that same pane before falling back
to the active pane only if the source pane no longer exists.

## What's New

- Command-V now follows the terminal pane that emitted the shortcut, matching
  the source-pane behavior already used by Command-C and Command-A.
- Multi-pane paste keeps working after pane switching, repeated paste events,
  and drag/rearrange layout changes.
- Added regressions for stale active-pane state, moved panes, missing source
  panes, and rapid paste requests across multiple terminal sessions.
- Retains the v1.0.14 source-pane Select All fix, the v1.0.13 source-pane copy
  fix, and the shared pasteboard bridge for pane-to-pane copy/paste.

## Why It Matters

Pane-to-pane copy/paste should feel boringly native: select or copy in one
terminal, move to another pane, paste, and keep working even after rearranging
the workspace. This release removes the stale active-pane edge case that made
paste feel random in some multi-pane sessions.

## Install

1. Download `gridOS-1.0.15-23-f7b51bc.dmg` from the `v1.0.15` GitHub release.
2. Open the DMG.
3. Drag `gridOS.app` into Applications.
4. If you installed an earlier copy, replace it.
5. Launch gridOS from Applications.

If macOS blocks the app, do not bypass Gatekeeper. Report it as a release issue.
