# gridOS 1.0.10

gridOS 1.0.10 is a stability and update-polish release for the native terminal
workspace. It keeps the 1.0.9 pane workflow and focuses on making the app feel
better over long sessions.

## What's New

- Added an in-app update affordance: when Sparkle finds a newer signed release,
  gridOS can surface an update badge in the header and route users into the
  standard update flow.
- Added clearer Software Updates controls for checking status, manually checking
  for updates, and updating when a newer release is available.
- Preserved current top-process CPU baselines across samples so the local
  metrics rail stays steadier during long-running sessions.
- Kept Command-T pane creation, Control-Tab pane traversal, drag-to-rearrange
  pane headers, AI Command Helper screenshot OCR, DeepSeek and xAI provider
  support, and Sparkle automatic updates.
- Shipped the update as a signed, notarized, stapled DMG and ZIP with refreshed
  Sparkle automatic-update metadata.

## Why It Stands Out

This release is less about adding a flashy surface and more about making the
daily-driver loop feel trustworthy. Update availability is visible in the app
instead of hidden in a menu, and the local process sampler keeps the baselines it
needs so terminal activity does not turn into distracting metric churn over
time.

## Install

1. Download `gridOS-1.0.10-18-26f01e7.dmg` from the `v1.0.10` GitHub release.
2. Open the DMG.
3. Drag `gridOS.app` into Applications.
4. If you installed an earlier copy, replace it.
5. Launch gridOS from Applications.

If macOS blocks the app, do not bypass Gatekeeper. Report it as a release issue.
