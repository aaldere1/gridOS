# gridOS 1.0.12

gridOS 1.0.12 is a cross-pane clipboard fix release. It makes terminal copy and
paste behave reliably when multiple panes are open, even when one pane is busy
or blocked.

## What's New

- Routed terminal Copy, Paste, and Select All through the active workspace pane
  instead of relying on stale AppKit responder focus.
- Copy now falls back to selected text in another terminal pane when the active
  pane has no selection.
- Paste now sends clipboard text directly into the active terminal PTY for
  app-driven paste commands.
- Command-T pane creation now activates and focuses the new pane more reliably.
- Pane layout, focus, move, resize, and font-size mutations now publish updates
  explicitly so SwiftUI does not miss nested state changes.
- Kept normal macOS copy/paste behavior for non-terminal controls by only
  intercepting shortcuts while a terminal view is the responder.

## Why It Matters

Multi-pane terminals only feel native when the pane you are looking at is the
pane that receives the command. This release fixes the confusing case where
copy/paste could appear to work in one pane but fail or land elsewhere when
another pane was active or blocked.

## Install

1. Download `gridOS-1.0.12-20-7b007d0.dmg` from the `v1.0.12` GitHub release.
2. Open the DMG.
3. Drag `gridOS.app` into Applications.
4. If you installed an earlier copy, replace it.
5. Launch gridOS from Applications.

If macOS blocks the app, do not bypass Gatekeeper. Report it as a release issue.
