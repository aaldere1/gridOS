Refactor terminal interactions from a single attached terminal/global notification model into an active-pane router with per-pane lifecycle control.

Purpose: Multi-pane UI must not broadcast copy/paste/clear/reset or Command Intelligence insertion to every terminal.
Output: `TerminalWorkspaceController`, extended `TerminalInteractionController`, pane-scoped `TerminalSurface`, and tests proving active-pane routing and cleanup.
</objective>
