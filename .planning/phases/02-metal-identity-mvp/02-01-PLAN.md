# Phase 2 plan - Metal identity MVP

## Goal

Prove gridOS can have a distinctive native visual identity without weakening terminal usability.

## Scope

This phase introduces the first Metal-backed background through `RenderCore`, wires coarse terminal activity into subtle render events, and keeps rendering isolated from terminal lifecycle behavior.

This phase does not implement multiple themes, custom terminal glyph rendering, metrics panels, LLM features, updater behavior, or production signing.

## Tasks

- [x] Add `RenderCore` visual identity model types.
- [x] Add deterministic procedural seed support.
- [x] Add a Metal-backed SwiftUI/AppKit host view.
- [x] Add a low-intensity shader that renders behind the terminal.
- [x] Add terminal activity events in `TerminalCore`.
- [x] Bridge terminal activity into render events from the app shell.
- [x] Keep rendering burst-driven or otherwise idle-conscious.
- [x] Add focused `RenderCore` tests.
- [x] Regenerate the Xcode project.
- [x] Run build/test verification.
- [x] Run app launch smoke verification.
- [x] Update architecture, release, roadmap, and state docs.

## Acceptance Criteria

- `RenderCore` owns the Metal surface and visual identity types.
- `TerminalCore` exposes only coarse activity events and does not depend on `RenderCore`.
- `GridOSApp` composes terminal and render modules without putting shader details in app code.
- The terminal remains the primary interactive surface.
- Renderer can draw an initial frame and react to terminal input/output events.
- Rendering idles when there is no activity burst.
- Unit tests cover deterministic seed/event behavior.
- `xcodebuild ... build test` passes.
- Smoke verification confirms the app launches, shell still starts, and terminal input still reaches the PTY.

## Risks

- Metal render work could increase idle CPU. Mitigation: make drawing burst-driven and document the current limit.
- SwiftUI state updates from terminal output could become too chatty. Mitigation: coalesce output activity before bridging to the app.
- Shader or MTKView setup can fail on unusual hardware. Mitigation: provide a non-crashing fallback background view.
