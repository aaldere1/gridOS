  <action>Create `CommandRiskClassifier` with `public func classify(_ command: String) -> CommandRiskAssessment`. `CommandRiskAssessment` must expose `level`, `reason`, and `policy`; levels are exactly `low`, `medium`, `high`, and `unknown`. `CommandRunPolicy` must include exact cases `canRun`, `requiresConfirmation`, and `insertOnly`. High-risk or unknown commands must never silently downgrade to low. Flag these D-17 classes as high risk: destructive filesystem operations (`rm -rf`, `find ... -delete`, `shred`, recursive chmod/chown), credential/keychain access (`security find-generic-password`, `security dump-keychain`, `Keychain`, `cat ~/.ssh`, `pbpaste |`), privilege escalation (`sudo`, `su`, `doas`), process killing (`kill`, `killall`, `pkill`), network transfer piped into shell (`curl ... | sh`, `curl ... | bash`, `wget ... | zsh`), package-manager install scripts (`brew install`, `npm install`, `pnpm install`, `pip install`, `curl ... install.sh`), and remote-service mutations (`git push`, `git reset --hard`, `kubectl apply/delete`, `docker rm`, `docker system prune`, `aws ... delete`, `gh repo delete`). Classify unknown or hard-to-review shell constructs as `unknown` with `insertOnly` or `requiresConfirmation`: command chaining `;`, `&&`, `||`, command substitution `$(` or backticks, redirects to privileged paths, and empty strings. Classify safe inspection commands such as `ls`, `pwd`, `git status`, `swift --version`, and `xcodebuild -version` as low with `canRun`; classify local project mutation like `git add`, `git commit`, and package install without remote shell as medium with `requiresConfirmation` unless already flagged high. Add explicit smoke-fixture coverage: `printf 'PHASE6_INSERT\n' > /tmp/gridos_phase6_insert.txt` must classify as low with `canRun` because it is the deterministic Phase 6 insert/run smoke command, while `rm -rf ~/tmp/gridos-test` must classify as high with `insertOnly` or confirmation. Tests must be fixture-driven and assert risk level, reason text, and policy for every listed class. Do not build a full shell parser; pattern rules and conservative unknown handling are the Phase 6 requirement.</action>
