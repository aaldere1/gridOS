# gridOS Design Context

## Target Audience

gridOS is for Mac developers, terminal-heavy operators, and local-AI power users who already have strong opinions about their tools. They live in shells, expect low-latency input, distrust hand-wavy AI automation, and will only adopt a new terminal surface if it feels faster, safer, and more memorable than their current setup.

## Core Use Cases

- Work in a real native terminal while keeping system state, visual identity, and command context visible.
- Use Command-K to turn intent, pasted output, or a failed command into a reviewed next move.
- Keep hosted AI optional, previewed, redacted, and guarded by local command policy.
- Recognize the machine as a distinct local command environment through the visual signature.
- Trust the app enough for daily work: launch fast, stay responsive, preserve panes, and never surprise-run commands.

## Brand Personality

Precise, cinematic, private, and confident. gridOS should feel like a Mac-native command deck built by people who respect terminals, not a toy sci-fi skin and not a generic AI wrapper. The tone is quiet power: guarded, local-first, technically literate, and production serious.

## Quality Bar

Flagship release candidate. No visible beta posture, no apologetic UI, no placeholder onboarding, and no feature that implies more capability than actually exists. The app should make a serious user think: this looks different, understands my workflow, and will not betray my terminal.

## Aha Moment

The first unforgettable moment is seeing a machine-specific visual signature around a usable terminal. The second is Command-K returning a clear, locally classified plan where the user can insert or run only after review.

## Design Principles

- Terminal trust comes first; visuals must frame the work, never compete with input.
- Command Intelligence should read like a guarded runbook, not a chat transcript.
- Local privacy is a product feature and should be visible without sounding defensive.
- Use tight, Mac-native density with cinematic hierarchy, not a marketing landing-page rhythm.
- Every production claim needs evidence: build, performance, signing, notarization, and launch smoke.
